ShelfSense API Manual Test Template
ShelfSense API 手动测试模板

========================================
1. Purpose / 用途
========================================

This file is a plain-text manual testing guide.
这个文件是纯文本的手动测试指南。

It is not meant to be executed directly.
它不是用来直接运行的脚本。

You can copy one section at a time into PowerShell.
你可以一次复制一段到 PowerShell 里执行。


========================================
2. Before Testing / 测试前准备
========================================

Step 1:
Start the API server first.
先启动后端服务。

Command:
uvicorn app.main:app --reload


Step 2:
Open another PowerShell window.
打开另一个 PowerShell 窗口。


========================================
3. Base URL / 基础地址
========================================

Copy:
$baseUrl = "http://127.0.0.1:8000"

Meaning / 含义:
- This is the local API address.
- 这是本地 API 地址。

What can be changed / 可以改什么:
- If your server runs on another port, change it here.
- 如果你的服务运行在别的端口，可以改这里。


========================================
4. Health Check / 健康检查
========================================

Copy:
Invoke-RestMethod -Method Get -Uri "$baseUrl/health"

Purpose / 用途:
- Check whether the server is running.
- 检查服务是否正常运行。


========================================
5. Register User / 注册用户
========================================

Copy:
$registerBody = @{
    username = "cyt"
    email = "cyt@example.com"
    password = "123456"
} | ConvertTo-Json

$user = Invoke-RestMethod `
    -Method Post `
    -Uri "$baseUrl/api/auth/register" `
    -ContentType "application/json" `
    -Body $registerBody

$user

Meaning / 含义:
- This creates a new user.
- 这会创建一个新用户。

What should be changed / 应该改什么:
- username
- email
- password

Important / 注意:
- email must be a valid email address.
- email 必须是合法邮箱。


========================================
6. Login / 登录
========================================

Copy:
$loginBody = @{
    email = "cyt@example.com"
    password = "123456"
} | ConvertTo-Json

$login = Invoke-RestMethod `
    -Method Post `
    -Uri "$baseUrl/api/auth/login" `
    -ContentType "application/json" `
    -Body $loginBody

$token = $login.access_token
$headers = @{ Authorization = "Bearer $token" }

$login

Meaning / 含义:
- This logs in and gets the bearer token.
- 这一步登录并获取 bearer token。

What should be changed / 应该改什么:
- email
- password

Important / 注意:
- These must match the registered account.
- 这里必须和刚才注册的账号一致。

17. Create Review / 创建评论

    -Method Post `
    -Uri "$baseUrl/api/reviews" `

What should be changed / 应该改什么:
- user_id
- book_id
- rating
- review_text

Important / 注意:
- rating must be between 1 and 5.
- rating 必须在 1 到 5 之间。
