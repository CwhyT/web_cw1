## 3. 考核任务

### (a) 最低技术要求

若想达到及格线（40 分及以上），你的 API 至少必须满足：

- 至少实现一个数据模型，并覆盖 CRUD 四项操作：Create、Read、Update、Delete，且与数据库相连。
- 你可以基于 RESTful API 设计；如果你认为更适合问题场景，也可以使用 GraphQL，但必须解释为何选择 GraphQL。
- 至少包含 4 个可通过 HTTP 请求访问的 API 端点。
- 能处理用户输入，并返回合适的 JSON 响应。如果使用其他格式，也必须在支持报告中说明理由。
- 按照行业惯例使用正确的状态码和错误码。
- 可以通过本地运行、Web 托管（例如 PythonAnywhere）或 Model Context Protocol (MCP) 服务器进行演示。

你可以用 Django 实现，也可以使用其他合适的技术栈，例如：

- Python FastAPI
- Node.js Express
- Go Fiber

你必须为你的技术选择和动机提供论证。

### (b) 项目选题示例

- 图书元数据与推荐 API：集成公开数据集（如 Goodreads 或 Google Books），并提供类型趋势、评分分布和个性化推荐等分析端点。
- 环境与城市气候统计 API：存储历史天气、空气质量和温度数据，使用户能够查询聚合指标、异常值和城市级趋势。
- 效率与习惯分析 API：跟踪任务、习惯、时间日志和完成模式，提供连续打卡计算、效率热力图和行为总结。

注 2：允许使用任何 SQL 数据库；除非你能在支持报告中给出明确理由，否则不应使用 NoSQL 数据库。

- 公共交通可靠性与延误分析 API：利用英国开放交通数据集，既提供用户上报事件的 CRUD，也提供路线可靠性评分、延误模式和时间表现总结等端点。
- 营养与食谱分析 API：基于公开食品数据集（如 USDA 或英国 FSA），支持食材/食谱 CRUD，并计算热量、宏量营养素、过敏原警告、成本或难度估计。
- 房地产市场与租房洞察 API：集成 ONS 或 Land Registry 数据，提供房源和地点的 CRUD，以及中位租金、可负担性指数或地区价格趋势分析。
- 文化活动与旅游洞察 API：使用博物馆、节庆和遗产地的公开数据，实现活动 CRUD，并提供热度指标、季节趋势和基于地理位置的推荐。
- 体育表现与比赛统计 API：基于公开体育数据集（如足球、板球或 F1），支持球队、球员、比赛的 CRUD，并提供排行榜、表现总结和胜率分析。

### (d) 使用 AI 处理数据集

鼓励你利用生成式 AI 工具帮助发现、处理并导入数据集到数据库。推荐流程如下：

#### (i) 数据集发现

用 AI 在线搜索你感兴趣领域中的数据集。你可以请 AI 推荐相关数据集、数据源或某一领域专门的数据仓库。

#### (ii) 下载与探索

下载一个有趣的数据集，并检查其结构、格式和内容，了解有哪些数据可用。

#### (iii) 生成导入脚本

把数据集的一部分提供给 AI，请它帮助你生成脚本，将数据导入并存储到 SQL 数据库中。这可以显著加快数据准备过程。

这种方法鼓励创造性探索，也体现了把 AI 作为生产力工具的有效用法。请记住按照第 2 节要求，在技术报告中声明你对 AI 工具的使用。
